Built for “just like vanilla” behavior

The output is structured around Minecraft’s standard music disc pattern: an item that points to a sound event, and a sound definition that points to your .ogg. In‑game it behaves like a normal disc: insert into a jukebox, it plays, and it’s obtainable via /give.

1) Import audio + optional models

Drop a folder’s worth of .ogg. If you also include .bbmodel, DiscForge will treat it as an optional “disc model source” you can later convert into JSON model files via your preferred pipeline. (The generator will package the raw BBModel alongside the mod template so nothing is lost.) 

# Recommended naming
mellow_night.ogg
mellow_night.bbmodel

# Becomes identifiers
modid:mellow_night
sound: modid:music_disc.mellow_night

2) Generate registrations + assets

We produce a clean, compile-ready structure: assets (sounds + lang + models) and code stubs (registries) for either Forge or Fabric style projects. You pick the mod ID, namespace, and a consistent naming scheme. 

// assets/<modid>/sounds.json
{
  "music_disc.mellow_night": {
    "sounds": [
      "records/mellow_night"
    ]
  }
}

3) Use /give + play in jukebox

Each disc item gets a stable identifier, so you can spawn it instantly: /give @p <modid>:<disc_id>. Once in your inventory, it’s a standard music disc item: insert into a jukebox and it plays your .ogg. 

# Give a disc
/give @p discforge:mellow_night

# Give multiple
/give @p discforge:mellow_night 3

# Vanilla-like usage: put in jukebox

FAQ

          
Practical answers for making discs behave like vanilla and keeping identifiers clean.

        

      


      

        

          

            
Will the discs work in a jukebox?

            

              Yes—when you compile the generated template into a mod, each disc is registered as a music disc item pointing to its sound event.
              In-game it inserts into a jukebox and plays like a vanilla disc.
            

          

          

            
Key requirements:
• Item type: MusicDisc
• Sound event: registered
• sounds.json: valid
• .ogg location: assets/<modid>/sounds/…

          

        


        

          

            
What about .bbmodel?

            

              This generator packages BBModel files and creates item model JSON stubs.
              Converting BBModel → Minecraft JSON varies by workflow, so DiscForge doesn’t pretend to “magically” convert models.
              It keeps the pipeline honest while still batching everything into one export.
            

          

          

            
Suggested workflow:
1) Export BBModel → mc item model JSON
2) Replace generated model stubs
3) Keep IDs the same

          

        


        

          

            
Do you upload my files?

            

              No. All processing happens locally in your browser session. The ZIP is generated client-side.
            

          

          

            
Privacy model:
• No network requests
• No external file hosts
• Export is a local download
