Drop files

Drag in many .ogg files (required) and .png and/or .json files (optional). We’ll match by filename and enforce rules:

  • If both .png and .json exist for the same base name, an error is logged and that disc is skipped.
  • If only .png exists, a flat item model is generated using that texture.
  • If only .json exists without a matching .png, an error is logged and that disc is skipped.
  • .ogg files are required for each disc.

Built for “just like vanilla” behavior

The output is structured around Minecraft’s standard music disc pattern: an item that points to a sound event, and a sound definition that points to your .ogg. In‑game it behaves like a normal disc: insert into a jukebox, it plays, and it’s obtainable via /give. The item model is either your uploaded .json or a generated flat model using your .png texture.

1) Import audio + textures/models

Drop a folder’s worth of .ogg files. Optionally add .png textures and/or .json item models. Rules:

  • If both .png and .json exist for the same base name, an error is logged and that disc is skipped.
  • If only .png exists, a flat item model is generated using that texture.
  • If only .json exists without a matching .png, an error is logged and that disc is skipped.
  • .ogg files are required for each disc. 
# Recommended naming
mellow_night.ogg
mellow_night.png
mellow_night.json (optional, but not with .png)

# Becomes identifiers
modid:mellow_night
sound: modid:music_disc.mellow_night

2) Generate registrations + assets

We produce a clean, compile-ready structure: assets (sounds + lang + models + textures) and code stubs (registries) for either Forge or Fabric style projects. You pick the mod ID, namespace, and a consistent naming scheme. 

// assets/<modid>/sounds.json
{
  "music_disc.mellow_night": {
    "sounds": [
      "records/mellow_night"
    ]
  }
}

3) Use /give + play in jukebox

Each disc item gets a stable identifier, so you can spawn it instantly: /give @p <modid>:<disc_id>. Once in your inventory, it’s a standard music disc item: insert into a jukebox and it plays your .ogg. 

# Give a disc
/give @p discforge:mellow_night

# Give multiple
/give @p discforge:mellow_night 3

FAQ

          
Practical answers for making discs behave like vanilla and keeping identifiers clean.

        

      


      

        

          

            
Will the discs work in a jukebox?

            

              Yes—when you compile the generated template into a mod, each disc is registered as a music disc item pointing to its sound event.
              In-game it inserts into a jukebox and plays like a vanilla disc.
            

          

          

            
Key requirements:
• Item type: MusicDisc
• Sound event: registered
• sounds.json: valid
• .ogg location: assets/<modid>/sounds/…

          

        


        

          

            
What about .png/.json?

            

              This generator packages PNG textures and JSON item models.
              If you upload both PNG and JSON for the same disc, it logs an error and skips that disc.
              If you upload only PNG, it generates a flat item model JSON referencing that PNG.
              If you upload only JSON without PNG, it logs an error and skips that disc.
            

          

          

            
Suggested workflow:
1) Upload .ogg + .png for each disc (recommended)
2) Optionally upload .json if you want a custom model (but not with .png)
3) Export and compile your mod

          

        


        

          

            
Do you upload my files?

            

              No. All processing happens locally in your browser session. The ZIP is generated client-side.
            

          

          

            
Privacy model:
• No network requests
• No external file hosts
• Export is a local download
